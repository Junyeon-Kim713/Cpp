# week 1-1
## 1, 몰랐던 용어 정리
- 니모닉 기호 : **사람이 기억하기 쉬운 형태로 만든 코드** / 복잡한 숫자 데이터에 사람이 이해할 수 있는 단어나 약어를 일대일로 대응시킨 것이 바로 니모닉 기호
## 2, 알고리즘 vs 코딩
- **알고리즘**. Ai가 대부분의 코딩을 대체하는 시점에 단순 작업이라고 볼 수 있는 코딩보다는 코딩을 하기위한 설계도인 알고리즘이 더 중요해졌다고 생각헙니다.
## 3, 클래스와 객체의 차이점
- 클래스는 사용자가 새로 정의한 사용자 정의 자료형이고, 객체는 그 클래스로 만든 실체이다.
- 따라서 클래스는 어떤 자동차(객체)를 만들기 위한 설계도이고, 객체는 그 설계도(클래스)로 만든 자동차(객체)라고 비유할 수 있다
## 4, 절차 지향 vs 객체 지향
- 절차 지향 : **어떤 순서로 일을 처리할 것인가**에 초점을 맞추는 방식
 1. 기능 중심으로 코드를 작성하며, 데이터와 함수가 분리
 2. 컴퓨터의 처리 구조와 유사해서 실행 속도가 매우 빠름
 3. 프로그램 규모가 커지면 코드가 복잡하게 얽혀 유지보수가 어려워질 수 있음.
- 객체 지향 : **누가 어떤 역할을 할 것인가**에 초점을 맞추는 방식
 1. 데이터와 그 데이터를 조작하는 함수를 하나의 클래스 안에 묶어서 관리
 2. 코드 재사용과 유지보수가 매우 편리
 3. 설계하는 데 시간이 오래 걸리고, 절차지향에 비해 실행 속도가 약간 느릴 수 있음
## 5, 라이브러리란?
- 프로그램을 만들 때 많이 쓰이는 기능을 미리 작성해놓은 코드, 목적파일로 존재
## 6, 라이브러리가 목적파일로 존재하는 이유
- **효율성**, 소스 코드를 기계어로 변환하는 컴파일 과정이 시간이 오래 걸리기 때문이다.
- **사용 편의성**, 목적 파일들로 묶인 라이브러리 파일로 받아서 쓰면 코드가 훨씬 깔끔해지고 관리가 쉬워짐
## 7, 링크 과정이 필요한 이유
- 기계어로 변환된 여러 소스 파일들을 합치고 라이브러리와 결합해야 하기 때문이다.
