# 접근지정자가 왜 필요한가?  
- 객체의 캡슐화를 위해 필요하다. 접근 지정자를 통해 클래스 외부로부터의 접근 허용여부를 지정해줄 수 있다.
# 생성자와 setter 멤버함수는 모두 멤버변수를 설정하는 기능이다. 차이점을 설명하라  
- 생성자는 객체를 생성할 떄 맨 처음 초기화를 위해 프로그램이 자동으로 호출하는 함수이고 setter 멤버함수는 사용자가 멤버변수의 값을 설정하고 싶을 때 직접 호출하는 함수이다.
# 객체지향형 언어의 특징인 캡슐화와 정보은닉을 설명하라.  
- 캡슐화는 데이터와 함수를 하나의 클래스 안에 모아놓고 외부로부터 안전하게 보호하는 기능이다.
- 정보은닉은 접근 지정자를 통해 멤버변수에 외부로부터의 직접적인 접근을 막고 멤버변수에 간접적으로 접근하는 멤버함수 정의해 멤버변수를 외부로부터 안전하게 보호하는 기능이다.
