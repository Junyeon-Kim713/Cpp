## 매크로상수 NULL이 무엇인지 조사하라, C/C++언어에서 정의의 차이, 정의된 헤더파일, 사용용도 등  
- 포인터가 아무 것도 가리키고 있지 않다는 의미를 가진 매크로 상수.
- C언어에서는 #define NULL ((void *)0)로 void*형으로 정의되어 컴파일러가 포인터 변수에 맞는 자료형으로 자동 형변환을 해줌.
- C++에서는 void*가 다른 포인터로 자동 형변환 되지 않기 때문에 #define NULL 0로 정의됨. C언어와 다르게 실제 정수 0으로 정의.
- 표준 라이브러리 헤더파일 안에 정의되어있음.
- 포인터 초기화, 유효성 검사, 자료구조의 끝 표시를 할 때 주로 사용됨.
## 널문자와 NULL의 차이를 설명하시오, 정의, 자료형, 메모리크기, 실제값, 사용용도 등
- 널문자는 문자열의 끝을 나타내는 제어문자이고, NULL은 아무런 주소도 가리키지 않는다는 매크로 상수이다.
- 자료형은 C언어 기준 int / void* 이고 C++ 기준 char / int, long 이다.
- 메모리 크기는  1바이트 / 4바이트,8바이트 이다.  
- 실제값은 아스키코드 0, 메모리 주소 0(C), 정수 0(C++) 이다.
- 사용용도는 문자열 배열의 끝 표시 / 포인터 초기화, 메모리 할당 확인 이다.
## 요즘은 NULL 대신에 nullptr을 사용하는데 nullptr은 무엇인지 조사하시오  
- C++에서 NULL은 C언어와 다르게 자동형변환이 안되기 때문에 오류를 막고자 실제 정수 0으로 정의되어있다. 따라서 함수오버로딩 등의 상황에서 컴파일러가 NULL을 주소가 아닌 정수로 처리해버리는 상황이 생겨
포인터에만 대입될 수 있는 주소형 nullptr을 만들었다.
