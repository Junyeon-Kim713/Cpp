# static 멤버변수는 클래스 선언부에 선언하고 외부에서 다시 선언하는 이유를 설명하라. 
- 스태틱 멤버 변수를 클래스 내부에서 정의하면 분할 컴파일할 때 클래스의 선언을 담 헤더파일을 포함한 모든 소스파일에서 스태틱 멤버변수의 메모리를 할당하기 때문에 링크 단계에서 에러가 발생한다. 따라서 클래스 바깥에서 정의를 해줌으로써 한 개의 소스파일에만 정의가 담기도록 한다.
# static 멤버함수의 클래스 구현부에 static 키워드를 붙이면 안되는 이유를 설명하라.
- 멤버함수를 정의할 때 static을 붙이면 해당 소스파일에서만 함수를 사용할 수 있기 때문에 분할 컴파일 시 에러가 발생한다.
